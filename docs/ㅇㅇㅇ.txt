비주얼 컨셉

색상: 밝은 베이지 배경 (#ecece5), 선명한 원색 사용
테두리: 없음
그림자: 3D 효과를 위한 박스 그림자 (4px 4px 0 rgba(0,0,0,0.3))
폰트: 영문은 --font-pixel을 따르고 한글은 --font-galmuri을 따르도록 함 

화면 레이아웃
좌상단: "사용자명"님 텍스트 (테두리 없음)
우상단: 5칸 게이지 바 (픽셀 박스 5개)
중앙 상단: 질문 텍스트 (큰 검은 글씨, 갈무리14폰트)
중앙 하단: 답변 선택 영역 (2x2 그리드 또는 유형별 레이아웃)
좌하단: 픽셀 캐릭터 (no-background sol png) > quiz-html과 레이아웃 동일
좌하단: 말풍선 > quiz-html과 레이아웃 동일
하단 전체: 픽셀 잔디 레이어 (초록색, 패턴) > quiz-html과 레이아웃 동일
최하단: 흙 레이어 (갈색, 픽셀 패턴)> quiz-html과 레이아웃 동일
장식요소 > quiz-html과 레이아웃 동일

🎯 5가지 퀴즈 유형 상세 명세
1️⃣ 드래그 앤 드롭 (Drag & Drop)
사용 사례: "개인정보 보호 관련 업무는 어느 부서의 책임인가?"
UI 구성:

상단: 점선 테두리의 드롭 타겟 박스 (150px 높이)
하단: 5개의 드래그 가능한 카드 (nes button )

예: "인사팀", "IT팀", "법무팀", "재무팀", "영업팀"



인터랙션:

드래그 시작: 카드 투명도 50%, 커서 grabbing
드롭 영역 호버: 연한 초록 배경 표시
드롭 완료:  300ms 후 자동 제출
정답: 별표 폭죽 효과 + 게이지 1칸 증가 (노란색)
오답: 화면 ±3px 흔들림 + 말풍선에 해설 타이핑

기술 요구사항:

HTML5 Drag & Drop API 사용
드래그 중 다른 요소 선택 불가
모바일 터치 지원 (선택사항)


2️⃣ 타이핑 문제 (Typing)
사용 사례: "다음 문장을 정확히 입력하세요"
UI 구성:

상단: 목표 문장 표시 (회색 박스, 큰 폰트)

예: "회사 기밀 정보는 외부에 유출할 수 없으며, 위반 시 징계 대상이다"

중앙: textarea 입력창 (최소 120px 높이)
하단: 진행률 바 (0-100%, 실시간 업데이트)

인터랙션:

입력 감지: 실시간 글자 수 / 목표 글자 수 계산
진행률: 현재길이 / 목표길이 * 100%
100% 달성: 효과음 재생
복사/붙여넣기: 완전 차단 (onPaste, onCopy, onCut, onContextMenu)
Enter 키: 제출 (입력 중일 때만)
정확도 검사: 띄어쓰기, 문장부호 포함 완전 일치

기술 요구사항:

우클릭 메뉴 비활성화
한글 입력 지원
100% 완료 시 0.5초 후 자동 제출


3️⃣ 빈칸 맞추기 (Fill in the Blank - 5지선다)
사용 사례: "직원은 출근 후 [  ]분 이내에 근태 시스템에 등록해야 한다"
UI 구성:

질문: 상단에 빈칸 표시 (예: [____])
선택지: 5개 버튼 (가로 배치, 자동 줄바꿈)

예: "10분", "20분", "30분", "60분", "15분"
각 버튼: 흰 박스, 검은 테두리, 최소 너비 180px


인터랙션:

호버: 연한 회색 배경
선택: 노란색 배경 + 굵은 글씨
클릭: 효과음 재생, 300ms 후 자동 제출
정답: 초록색 배경
오답: 빨간색 배경 + shake 애니메이션

기술 요구사항:

키보드 내비게이션 지원 (화살표 키)
한 번 선택하면 다른 선택 불가 (제출 전까지)


4️⃣ OX 퀴즈 (말풍선 힌트)
사용 사례: "회사 노트북을 개인 용도로 사용해도 된다"
UI 구성:

중앙: "O" (왼쪽) VS (중간) "X" (오른쪽)

O와 X: 120px 초대형 글자, 배경/테두리 없음
흰색 외곽선 (text-shadow 8방향)


말풍선: 캐릭터 위에 위치

인터랙션:

O에 마우스 호버:

정답이면: 말풍선에 큰 👍 (60px)
오답이면: 말풍선에 큰 👎 (60px)


X에 마우스 호버:

정답이면: 말풍선에 큰 👍
오답이면: 말풍선에 큰 👎


마우스 떠남: 말풍선 숨김
클릭: 선택 상태 표시, 300ms 후 자동 제출
정답: O 또는 X가 초록색으로 변경
오답: O 또는 X가 빨간색으로 변경

기술 요구사항:

onMouseEnter/onMouseLeave 이벤트
손가락 이모지에 drop-shadow 효과
호버 시 scale(1.1) 애니메이션


5️⃣ 틀린 부분 찾기 (Find Error)
사용 사례: "다음 문장에서 틀린 부분을 찾으세요"
UI 구성:

문장: 큰 흰 박스에 32px 폰트

예: "직원은 출근 시 회사 보안카드를 착용하고, 퇴근 시에도 반드시 패용해야 한다"


밑줄 단어: 5개 (클릭 가능, 밑줄 표시)

예: 출근(O), 보안카드(O), 착용(O), 퇴근(O), 패용(X - 정답)


하단: 힌트 텍스트 "(밑줄 친 단어를 클릭하세요)"

인터랙션:

호버: 노란색 하이라이트
선택: 금색 배경 + 굵은 글씨
클릭:300ms 후 자동 제출
정답: 초록색 배경
오답: 빨간색 배경 + shake 애니메이션

기술 요구사항:

동적으로 밑줄 단어를 span으로 감싸기
한 번 선택하면 다른 단어 선택 불가 (제출 전까지)


🎮 핵심 게임 요소
⭐ 5칸 게이지 시스템 (우상단)
구조:

5개의 정사각형 픽셀 박스 (35x35px)
검은 테두리로 빈 박스 표시

상태별 색상:

빈 칸: 검은 테두리만
정답 (첫 시도): 노란색 (#ffd700) 채우기
정답 (재시도): 회색 (#808080) 채우기
럭키드로우 당첨: 특수 아이콘 표시
럭키드로우 탈락: 특수 아이콘 표시

애니메이션:

정답 시: 해당 칸 위치에서 별표 폭죽 효과 (20개 별, 360도 방향)


🎆 별표 폭죽 효과
트리거: 첫 시도에 정답 맞춤
동작:

현재 문제 번호의 게이지 칸 위치 계산
그 위치를 중심으로 20개의 "⭐" 생성
360도 방향으로 퍼지는 애니메이션 (80-140px 거리)
1.5초 후 자동 제거

CSS:

position: fixed
animation: firework-explode 1.5s ease-out
--tx, --ty CSS 변수로 각도별 이동 거리 제어

🔧 기술 요구사항
HTML 구조
quiz-container
├── user-title (좌상단 사용자명)
├── gauge-bar (우상단 5칸 게이지)
│   └── gauge-box × 5
├── question (질문 텍스트)
├── answers (답변 영역, 동적 렌더링)
├── next-question-btn (NEXT 버튼, 숨김)
└── explanation-bubble (말풍선, 숨김)

ground
├── grass-layer
└── dirt-layer

decorations
├── character
├── balloon-orange
├── balloon-blue
├── tree
├── sunflower
└── ... (기타 장식)

fireworks-container (폭죽용)
CSS 핵심 기법

픽셀 렌더링: image-rendering: pixelated
픽셀 아트: box-shadow 다중 사용
테두리: 5px solid #000
그림자: 4px 4px 0 rgba(0,0,0,0.3)
애니메이션: @keyframes + animation

JavaScript 로직
주요 함수:
- init(): 초기화
- loadQuestion(): 문제 로드
- renderQuestion(type): 문제 유형별 렌더링
  - renderDragDrop()
  - renderTyping()
  - renderFillBlank()
  - renderOX()
  - renderFindError()
- handleSubmit(): 답안 제출
- showFeedback(isCorrect): 정답/오답 처리
- triggerFireworks(): 폭죽 효과
- updateStarGauge(index, status): 게이지 업데이트
- typeWriterEffect(element, text): 타이핑 효과
